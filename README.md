# CS50P — Introduction to Programming with Python

Solutions to Harvard's [CS50P](https://cs50.harvard.edu/python/) problem sets — the Python foundations phase of my self-study path toward backend / software engineering roles.

## Structure

Flat layout — one file (or folder, for problems that ship with starter data/tests) per problem, named to match the course's own problem names:

## Approach

- Solve independently first, then compare against `check50` output rather than the other way around
- Re-read `style50` feedback even after a pass — clean style now saves relearning habits later
- Treat each problem set's edge cases (empty input, bad input, unexpected types) as part of the problem, not an afterthought
